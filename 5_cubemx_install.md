# Установка CubeMX


  CubeMX - графический инструмент для настройки микроконтроллеров STM32 и гене-
рации кода инициализации.


## Скачиваем по ссылкам ниже нужную версию программы и репозиториев:

[Облако mail.ru доброго человека](https://cloud.mail.ru/public/2i19/Y4w8kKEiZ)

[Репозиторий на github похоже официальный, но тут самого CubeMX нету](https://github.com/orgs/STMicroelectronics/repositories?type=all)

## Создаём директории в `opt`:

  Создаём две директории в `/opt` под суперпользователем, так как доступ в 
директорию `/opt` имеет только суперпользователь. Задача - создать внутри неё
директорию, где нашему пользователю будет разрешено работать. Поэтому, для
установки патчей даём пользователям доступ на запись в директорию.

```bash
mkdir -p /opt/cubemx/repository         # создание директорий

chown $USER /opt/cubemx    # изменения владельца на нужного
                                        # пользователя
chmod -R 775 /opt/cubemx                # установка прав на полный доступ
                                        # указанному пользователю
                                        # чтение и исполнение всем остальным
                                        # без разрешения на запись
```

## Установка:
  Распаковываем и запускаем установщик, во время установки следим, чтобы
программа установилась в созданные нами директории.



**P.S.:** Можно было бы сразу две папки создать и сразу двум папкам права
назначить, но тогда CubeMX при установке ругается, что папка не является
пустой и не даёт продолжить установку.


## Запускаем CubeMX через меню приложений. Заходим в меню 'Help'
    
* `Help` -> `User Preferences` убираем единственную галочку и нажимаем `Save`

* `Help` -> `Connection & Updates`
тут прописываем путь к репозиторию `/opt/cubemx/repository`
и ставим выбор в чек боксе на `Manual Check`, что `CubeMX` не беспокоил
сообщениями о выходе новых версий.
Нажимаем `OK`

## Редактируем ярлык в меню программ текстовым редактором


```bash
nvim .local/share/applications/com-st-microxplorer-maingui-STM32CubeMX.desktop
```

добавляем строчку:

```ini
Categories=Development;
```

или выполняем команду

```bash
echo "Categories=Development;" | tee -a ~/.local/share/applications/com-st-microxplorer-maingui-STM32CubeMX.desktop
```


  Это переместит ярлыку программы в меню приложений в категорию `Development`
(или `Разработка`, в зависимости от дистрибутива), где уже находится `VSCode`.
