# Описание установки консольного текстового редактора neovim

## Предисловие:
  
  В своё время на ноутбуке решил оставить только систему linux. На тот момент
это была `Ubuntu Mate`.

  На ней начал изучать `python`, `C` и сам `linux`. Начал писать для себя
заметки и уроки, потому что со временем, без практики информация забывается,
а так можно было быстро к ним вернуться.

  В этот момент почему-то захотелось попробовать консольный текстовый редактор.
Из основных, которые используют в системе сразу обычно установлен `nano`, `vi`,
также часто используют `mcedit`, который устанавливается в месте
с `midnight commander` (он же `mc`).

  Начал работать в `nano`, но быстро понял, что использовать его неудобно.
Первое, что бросилось в глаза - это отсутствие переноса строк, за этим постоянно
приходилось следить. И вообще навигация по текстовому файлу в консольном режиме
не самое удобное дело, но было интересно. В результате через некоторое время
открыл для себя `vim`.

  Ранее, примерно году в 2008, когда меня появился более менее нормальный интер-
нет и я первый раз установил linux к себе на компьютер, я его запускал, но
ничего не понял. Было проще использовать nano для редактирования файлов
конфигурации, когда не было возможности запустить графический текстовый
редактор. На тот момент для меня это показалось чем-то диким (редактор `vim`),
для гиков, использовать его.

  Но после просмотра нескольких видео по этой теме на тогда ещё доступном
`youtube` заинтересовался.

  В результате сегодня большая часть заметок, которые требуют внимания, и
вдумчивого написания сделаны в `nvim` (современная версия `vi`).

  Полностью написана курсовая по информатике, написано ПО по работе, которые я
как-то пересылал, написан курс по `git` в формате `markdown`, который лежит в
репозитории на `GitHub`. И в целом его дальнейшее изучение продолжается, он
очень комплексный, и его применение имеет много плюсов. Например, он позволяет
быстро писать программы даже на слабых компьютерах, заставляет думать иным
образом, что помогает концентрации на задаче. Позволяет работать без 
использования мышки (например, лёжа в кровати), для работы достаточно
стандартной клавиатуры.

  Использовать его или нет выбор каждого, но рассмотрим установку.


## Установка

  Выполняем команду на установку с помощью стандартного менеджера пакетов `apt`
(или `apt-get`). В зависимости от предпочтений и установленной системы.

```bash
sudo apt install neovim
```

  В репозиториях `debain` ветки дистрибутивов называется `neovim`, однако запуск
программы с помощью `nvim`.

 Всё готово, но надо настроить. Для настройки необходимо установить
дополнение `plug-install`.
  Оно представляет из себя один скриптовый файл, в котором прописаны некоторые
дополнительные функции для работы с быстрой установкой и обновлением дополнений.
Устанавливается путём скачивания этого файла из репозитория разработчика
плагина на `GitHub` командой в консоли:

```bash
sh -c 'curl -fLo "${XDG_DATA_HOME:-$HOME/.local/share}"/nvim/site/autoload/plug.vim --create-dirs https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim'
```

Длинная команда, это скрипт для `shell`. Он создаёт папки по пути
```~/.local/share/nvim/site/autoload```
и скачивает туда файл `plug.vim`, который и является нужным нам дополнением.

`~` в начале адреса - это директория пользователя,
    то есть `/home/[Имя_пользователя]`

  Теперь нужно создать файл конфигурации. Установленное дополнение будет
работать (скачивать, обновлять, запускать) дополнения для `nvim`, указанные в
файле конфигурации в блоке


```ini
call plug#begin()

" List your plugins here

call plug#end()
```

## Настройка

  Настройку для использованных в этом списке плагинов необходимо прописывать
ниже этого блока, так как `plug-install` запустит их после выполнения этого
блока.

  Свой базовый файл конфигурации с комментариями блэкджеком и остальным,
приложу ниже.

Его нужно скопировать в `~/.config/nvim/`,
    то есть `/home/$USER/.config/nvim/`.

Сам файл называется `init.vim`,
    то есть полный путь
            `/home/$USER/.config/nvim/init.vim`.

Есть более продвинутый вариант конфигурация на `lua`, там нужно создавать файл
    `init.lua`.
Больше возможностей, сложнее, но мне достаточно `init.vim`.

## Использование

  Запустить `nvim`. Написать команду:
```
    :PlugInstall (она будет вводиться внизу экрана в самой нижней строке)
```
   Подождать, пока оно там всё сделает чего надо.

  Выйти из `nvim`, набрав команду туда же в нижнюю строку
```
    :q
```

  Запустить заново `nvim` уже полностью настроенный.

  То есть, если коротко:
    - ставим `neovim`:
    - копируем файл с `plug-install` длинной командой из интернета (при пере-
        установке системы он уже будет там);
    - копируем файл конфигурации в нужную папку (при переустановке системы
        он уже будет там);
    - запускаем `nvim`, в соглашаемся и пропускаем все ошибки;
    - пишем команду `:PlugInstall`, ждём когда всё будет готово;
    - перезапускаем `nvim`.
